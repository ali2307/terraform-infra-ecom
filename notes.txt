SECURITY PLAN (Important Part)
=================================

Enterprises care about:


State	       -       Remote backend (S3 + DynamoDB lock)
Secrets	       -       No secrets in code
Access	       -       IAM roles, least privilege
Branches	   -       Protected branches
Pipeline	   -       Plan → Approval → Apply
Scanning	   -       tfsec / checkov
Isolation	   -       Separate env folders
Auditing	   -       Git history + state versioning

STEP 1 — Remote Backend (Enterprise MUST)

Create S3 + DynamoDB manually once:

aws s3api create-bucket --bucket ecommerce-tf-state-uae-20260128 --region me-central-1 --create-bucket-configuration LocationConstraint=me-central-1

aws dynamodb create-table `
--table-name terraform-lock `
--attribute-definitions AttributeName=LockID,AttributeType=S `
--key-schema AttributeName=LockID,KeyType=HASH `
